## Bug #001：游客模式下新建会话后未显示游客标签

| 字段 | 内容 |
|------|------|
| Bug ID | BUG-001 |
| 标题 | 【游客模式】新建会话后顶部"游客模式"标签短暂消失 |
| 模块 | AI 对话 |
| 严重程度 | 低 (Low) |
| 优先级 | 低 (Low) |
| 环境 | Chrome 125 / Windows 11 |
| 版本 | v1.0 |

### 前置条件
未登录状态，处于 /chat 页面

### 复现步骤
1. 未登录访问 /chat，确认顶部显示"游客模式"标签
2. 点击左侧「新建对话」按钮
3. 观察顶部导航变化

### 实际结果
新建会话瞬间，"游客模式"标签消失约 1 秒后重新出现

### 预期结果
"游客模式"标签应始终显示，不应闪烁

## Bug #002：注册时用户名超长未做前端校验

| 字段 | 内容 |
|------|------|
| Bug ID | BUG-002 |
| 标题 | 【注册】用户名输入 50 位字符时无前端拦截，提交后后端报错 |
| 模块 | 用户注册 |
| 严重程度 | 中 (Medium) |
| 优先级 | 中 (Medium) |
| 环境 | Chrome 125 / Windows 11 |

### 复现步骤
1. 进入注册页面
2. 用户名输入 50 个字母（如 aaaaaaaaa...）
3. 填写其他合法信息
4. 点击注册

### 实际结果
表单通过前端校验，提交后后端返回错误或保存异常

### 预期结果
用户名输入框应有长度限制提示（如"用户名不超过 20 位"）

### 建议
在 `Register.vue` 的 `registerRules` 中增加 `max: 20` 校验规则

## Bug #003：未登录直接访问 /verify 白屏

| 字段 | 内容 |
|------|------|
| Bug ID | BUG-003 |
| 标题 | 【权限】未登录直接访问 /verify 页面白屏 |
| 模块 | 路由权限 |
| 严重程度 | 高 (High) |
| 优先级 | 高 (High) |
| 环境 | Chrome 125 |

### 复现步骤
1. 清除所有 localStorage 数据
2. 浏览器直接访问 /verify

### 实际结果
页面白屏，控制台报错，无任何内容渲染

### 预期结果
应正常显示邮箱验证页面，或跳转到登录页

### 根因分析
路由守卫中 /verify 已列入 `guestPages`，可能是验证页面组件依赖了用户状态但未做空值判断
