## 1. 测试目标

对「奈克瑟 NEXUS」AI 对话平台进行功能测试和 UI 测试，确保核心功能可用、用户体验良好，识别并记录缺陷。

## 2. 测试范围

### 2.1 纳入测试的功能模块

| 模块 | 优先级 | 说明 |
|------|--------|------|
| 用户注册/登录 | P0 | 最核心流程，阻塞其他功能 |
| AI 对话 | P0 | 产品核心功能 |
| 游客模式 | P0 | 未登录用户的体验 |
| AI 图片生成 | P1 | 重要辅助功能 |
| 知识库管理 | P1 | 文件上传与管理 |
| 导航与权限 | P0 | 访问控制正确性 |
| 页面 UI 展示 | P2 | 样式与适配 |

### 2.2 不纳入测试范围

- 后端 API 内部逻辑（纯白盒，本次仅做端到端验证）
- 性能测试（并发/压力，需专门环境）
- 安全渗透测试（SQL 注入、XSS 深度测试）
- AI 模型回复内容的准确性（属于模型能力，非应用层问题）

## 3. 测试策略

### 3.1 功能测试
- 按测试用例逐条执行
- 覆盖正常流程（Happy Path）和异常流程
- 边界值测试：输入长度、文件大小、次数限制

### 3.2 UI 测试
- 页面布局完整性
- 移动端适配（375px 宽度）
- 浏览器兼容性（Chrome / Edge / Firefox）

### 3.3 探索性测试
- 在功能测试之外，自由操作 30 分钟
- 关注边界情况和异常操作组合

## 4. 测试通过标准

- P0 级用例 100% 通过
- 所有用例通过率 ≥ 90%
- 无 P0/P1 级未修复 Bug
- 探索性测试未发现阻塞性问题

## 5. 测试暂停/恢复条件

### 暂停条件
- 环境不可用（后端服务无法启动）
- 核心功能（AI 对话）完全不可用
- 阻塞性 Bug 导致大部分用例无法执行

### 恢复条件
- 阻断问题已被修复并验证
- 环境恢复正常
